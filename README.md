# High-Load Network Telemetry Service

Высокопроизводительный микросервисный комплекс на Go для сбора, анализа и мониторинга телеметрии сетевого оборудования в режиме реального времени. Архитектура построена на базе событийно-ориентированного подхода (Event-Driven Architecture).

## Архитектура системы

Система разделена на независимые слои, изолированные в Docker-контейнерах:

1. Telemetry Collector (Go): Имитирует работу распределенной сети роутеров, генерирует метрики нагрузки (CPU, Memory, Status) и асинхронно отправляет их в брокер сообщений.
2. Apache Kafka (KRaft mode): Распределенная шина данных, обеспечивающая высокую пропускную способность и гарантированную доставку сообщений.
3. Telemetry Analyzer (Go): Потоковый процессор. Читает сообщения из Kafka, валидирует JSON, выявляет критические алерты (нагрузка более 85%), обновляет оперативный кэш состояния и сохраняет историю.
4. Telemetry API (Go / gRPC): Высокопроизводительный gRPC-сервер, предоставляющий методы для внешних клиентов: быстрый статус из Redis и исторические метрики из PostgreSQL.
5. Хранилища данных:
   * Redis 7 (Alpine): Оперативный InMemory-слой для мгновенного чтения текущего статуса устройств.
   * PostgreSQL 15 (Alpine): Реляционная БД для надежного хранения аналитической истории метрик.
6. Наблюдаемость (Observability):
   * Prometheus: Автоматически собирает внутренние бизнес-метрики анализатора (кол-во обработанных сообщений, сгенерированные алерты).
   * Grafana: Визуализирует метрики на интерактивных дашбордах.

## Технологический стек
* Язык разработки: Go (версия 1.26+)
* Шина данных: Apache Kafka (github.com/segmentio/kafka-go)
* API Протокол: gRPC + Protocol Buffers (google.golang.org/grpc)
* Базы данных: PostgreSQL (database/sql + lib/pq), Redis (go-redis/v8)
* Миграции: golang-migrate
* Мониторинг: Prometheus client, Grafana
